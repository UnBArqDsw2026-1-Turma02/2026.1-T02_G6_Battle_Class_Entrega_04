## Demo de Reutilização (`demo:reuso`)

O artefato executável `demos/demo-reuso.ts` demonstra **o ato de reutilizar** em quatro
cenas progressivas. Um único comando o aciona:

```bash
npm install
npm run demo:reuso
```

### Cena 1 — Reuso caixa-preta (Registry + Factory Method)

O cliente instancia o `CreatorRegistry` padrão e solicita um inimigo e uma questão **sem
conhecer** qual `ConcreteCreator` está por trás.

```typescript
// demos/demo-reuso.ts (Cena 1)
import { CreatorRegistry } from '../src/server/factories/index.js';

const registry = new CreatorRegistry();

// Cliente só conhece a interface — ignora o ConcreteCreator
const inimigo = registry.inimigoPara('matematica').spawn(2);
console.log('[Cena 1] Inimigo:', inimigo.render());

const questao = registry.questaoPara('ENEM').criarQuestao('Texto da questão base');
console.log('[Cena 1] Questão:', questao.enunciado);
```

**O que evidencia:** reuso caixa-preta por composição — o cliente depende apenas do
contrato (`CreatorRegistry`), não de nenhuma classe concreta.

---

### Cena 2 — Substituição black-box sem editar nada (OCP por composição, 0 linhas no domínio)

Um `InimigoCreator` alternativo é criado localmente e injetado via construtor do
`CreatorRegistry`. O cliente continua intocado; só a configuração muda.

```typescript
// demos/demo-reuso.ts (Cena 2)
import { InimigoCreator } from '../src/server/factories/inimigo/InimigoCreator.js';
import type { Inimigo } from '../src/shared/types.js';

// Creator alternativo: reutiliza spawn() por herança, só muda factoryMethod()
class InimigoMatematicaEliteCreator extends InimigoCreator {
  protected override factoryMethod(): Inimigo {
    return { hp: 200, velocidade: 1.5, materia: 'matematica', render: () => '👾 Elite' };
  }
}

const registryElite = new CreatorRegistry({
  matematica: new InimigoMatematicaEliteCreator(),
  historia:   registry.inimigoPara('historia').__creator,   // reusa o default
  biologia:   registry.inimigoPara('biologia').__creator,
});

const inimigoElite = registryElite.inimigoPara('matematica').spawn(2);
console.log('[Cena 2] Inimigo elite (substituição black-box):', inimigoElite.render());
// Nenhum arquivo de serviço ou cliente foi editado.
```

**O que evidencia:** o Registry é **aberto para substituição** sem alterar o domínio —
coração da prova de reuso limpo. Tipa corretamente pois as chaves respeitam `Materia`.

---

### Cena 3 — Ponto de extensão: nova família `InimigoQuimica*` (custo honesto = 1 linha)

`Materia` é um union **fechado**. Para adicionar uma matéria nova, o custo real é
**1 linha em `shared/types.ts`** + 2 arquivos novos. Nenhum serviço cliente é alterado.

```typescript
// src/shared/types.ts  ← única linha editada no domínio
export type Materia = 'matematica' | 'historia' | 'biologia' | 'quimica'; // +1 linha

// src/server/factories/inimigo/InimigoQuimica.ts  ← arquivo novo
export class InimigoQuimica implements Inimigo {
  readonly materia: Materia = 'quimica';
  constructor(readonly hp: number, readonly velocidade = 1.0) {}
  render() { return `⚗️ InimigoQuimica hp=${this.hp}`; }
}

// src/server/factories/inimigo/InimigoQuimicaCreator.ts  ← arquivo novo
export class InimigoQuimicaCreator extends InimigoCreator {
  protected override factoryMethod(): Inimigo {
    return new InimigoQuimica(80); // spawn() herdado escala o HP por onda
  }
}
```

```typescript
// demos/demo-reuso.ts (Cena 3)
import { InimigoQuimicaCreator } from '../src/server/factories/inimigo/InimigoQuimicaCreator.js';

const registryComQuimica = new CreatorRegistry({
  matematica: new InimigoMatematicaCreator(),
  historia:   new InimigoHistoriaCreator(),
  biologia:   new InimigoCreatorBiologia(),
  quimica:    new InimigoQuimicaCreator(),   // nova família
});

const inimigoQuimica = registryComQuimica.inimigoPara('quimica').spawn(2);
console.log('[Cena 3] Nova família (extensão + herança):', inimigoQuimica.render());
// VestibularServiceFacade, PartidaFacade, TowerDefenseService → INTOCADOS
```

**O que evidencia:** OCP preservado na camada de aplicação. O **diff** deste commit mostra:
apenas arquivos novos + 1 linha no `shared/` — zero alteração em clientes. Este é o senso
crítico em ação: reuso caixa-preta tem limite quando o contrato é um union fechado; o
custo do ponto de extensão é **explícito e honesto**, não escondido.

### Cena 4 — Reuso de estilo arquitetural (camadas observáveis)

Esta cena distingue **padrão GoF** (Facade) de **estilo arquitetural** (camadas da E02) e
torna o estilo **visível na saída**, não apenas afirmado.

```typescript
// demos/demo-reuso.ts (Cena 4)
import { VestibularServiceFacade } from '../src/server/facades/VestibularServiceFacade.js';

// Log que mostra a travessia das camadas conforme arquitetura da E02
console.log('[Cena 4] Interface → chamando VestibularServiceFacade...');
const facade = new VestibularServiceFacade();  // Aplicação (Facade)

const questoes = await facade.obterQuestoes({ banca: 'UNB', quantidade: 1 });
//                           ↑ Domínio: CreatorRegistry/Creator
//                                       ↑ Infra: mock injetado (sem rede)

console.log('[Cena 4] Camadas percorridas: Interface → Aplicação → Domínio → Infra');
console.log('[Cena 4] Questão obtida:', questoes[0].enunciado);
```

**O que evidencia:** o enunciado exige "padrões **E** estilos arquiteturais". A Facade é o
**padrão**; as camadas (Interface → Aplicação → Domínio → Infra) são o **estilo**, herdado
da modelagem da Entrega 02. A saída no terminal torna as camadas observáveis, não só
descritas.

## Como executar (critério objetivo de "RODANDO")

```bash
# 1. Instalar dependências
npm install

# 2. Verificar tipos
npm run typecheck     # deve retornar sem erros

# 3. Rodar o demo (4 cenas)
npm run demo:reuso    # imprime Cenas 1–4; Cena 3 mostra nova família sem editar cliente

# 4. Rodar toda a bateria de testes (inclui reuso.test.ts)
npm test              # todos os testes devem passar (verde)
```


## Teste automatizado (`tests/reuso.test.ts`)

O teste cobre os 4 caminhos exigidos pelo plano:

| Caso | O que verifica |
|---|---|
| **(a)** Registry default resolve os 3 tipos | `inimigoPara('matematica').spawn(1)` retorna `Inimigo` com `hp > 0` |
| **(b)** Substituição black-box (Cena 2) | Creator alternativo injetado é devolvido sem `instanceof` do cliente |
| **(c)** `spawn` herdado escala HP (`hpBase * (1 + onda * 0.15)`) na família nova | `InimigoQuimicaCreator` — `expect` fixado no `hp` base real |
| **(d)** Caminhos de erro | `inimigoPara('inexistente')` → `MateriaInexistenteError`; `questaoPara('XPTO')` → `EntradaInvalidaError`; `criarQuestao('')` → `EntradaInvalidaError`; `spawn(-1)` → `Error` puro |
